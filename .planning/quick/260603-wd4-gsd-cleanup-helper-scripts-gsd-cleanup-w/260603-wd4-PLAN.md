创建 `scripts/gsd-cleanup-worktree.sh` (项目根下的新目录,需要 `mkdir -p scripts`)。脚本结构:

1. **Shebang + strict mode**:
   - `#!/usr/bin/env bash`
   - `set -euo pipefail`

2. **用法与入参**:
   - `usage() { echo "Usage: $0 <manifest-path>"; exit 2; }`
   - `MANIFEST="${1:-}"`; 空则调 `usage`。
   - 文件不存在则 `echo "[gsd-cleanup] manifest not found: $MANIFEST" >&2; exit 2`。

3. **辅助函数**:
   - `read_manifest()`:用 `jq` 解析 manifest 的 `.worktrees[]` 数组(假设 `jq` 可用,与 hook 一致;若不可用则降级为 python3 内联)。每条 entry 至少需要 `worktree_path` 和 `branch`;其他字段透传。
   - `worktree_basename()`:`basename` 一次;`worktree_locked_marker()`:输出主仓 `.git/worktrees/<basename>/locked` 绝对路径,主仓根由 `git rev-parse --show-toplevel` 解析。

4. **Step 1: 救未跟踪的 *.md**:
   - 对 manifest 里每个 `worktree_path`:
     - 提取 `<worktree>/.planning/quick/<task_dir>/*-SUMMARY.md`(其中 `<task_dir>` 形如 `260603-*`)。用 `find "$worktree_path/.planning/quick" -type d -maxdepth 1 -mindepth 1 2>/dev/null` 拿到 task 目录列表,再 `find` 每个 task 目录下的 `*-SUMMARY.md`。
     - 对每个 SUMMARY.md:推导主仓对应路径 `<repo_root>/.planning/quick/<task_dir>/<file>`(`task_dir` = SUMMARY 父目录名)。
     - 若目标不存在 OR 与源不同(`! cmp -s "$src" "$dst"`),`mkdir -p "$(dirname "$dst")"` 再 `cp "$src" "$dst"`,并 `echo "[gsd-cleanup] rescued $src -> $dst"`。源/目标一致则跳过,无 echo。
   - 主仓根用 `REPO_ROOT=$(git rev-parse --show-toplevel)` 在脚本启动时解析一次。

5. **Step 2: 解除 worktree locked 标记**:
   - 对每个 `worktree_path`:`basename` -> `LOCK="$REPO_ROOT/.git/worktrees/$BASENAME/locked"`;`if [ -e "$LOCK" ]; then rm -f "$LOCK" && echo "[gsd-cleanup] removed locked marker: $LOCK"; fi`。`rm -f` 在文件不存在时不会报错,这样幂等。

6. **Step 3: 移除 untracked 阻塞文件**:
   - 对每个 `worktree_path`:
     - `NM="$worktree_path/node_modules"`
     - 若 `-L "$NM"`(symlink):`rm "$NM" && echo "[gsd-cleanup] removed symlink: $NM"`。
     - 若 `-d "$NM"`(目录)且非 symlink:不递归删(危险);改用 `if [ -z "$(find "$NM" -maxdepth 1 -type l 2>/dev/null)" ]` 检测内部 symlink——若空则 `echo "[gsd-cleanup] WARNING: $NM is a real directory, not auto-removed (manual cleanup needed)"` 写到 stderr,继续;不阻断。
     - 其他情况(不存在、不是 symlink 不是 dir):跳过。

7. **Main step**:
   - `echo "[gsd-cleanup] step 4: invoking gsd-sdk cleanup-wave"`
   - `if ! gsd-sdk query worktree.cleanup-wave --manifest "$MANIFEST"; then RC=$?; ...` 这里 **不能** 用 `set -e` 中断,要在 if 里捕获并继续。
   - 捕获 gsd-sdk 输出的 stderr(整段命令输出包含 stdout+stderr;用 `2>&1` 重定向到一个临时变量):
     ```bash
     OUTPUT=$(gsd-sdk query worktree.cleanup-wave --manifest "$MANIFEST" 2>&1) || GSD_RC=$?
     echo "$OUTPUT"
     ```
   - 若 `GSD_RC != 0`:
     - 扫描 `OUTPUT` 看是否含 `branch_contains_deletions` / `merge_failed` / `base_mismatch` 任一关键字。
     - 命中则额外输出多行提示到 stderr:
       ```
       [gsd-cleanup] DETECTED: branch_contains_deletions (or merge_failed / base_mismatch)
       [gsd-cleanup] HINT: this branch's merge produces file deletions that gsd-sdk guards against.
       [gsd-cleanup] Workaround:
       [gsd-cleanup]   - Manual merge:  git merge --no-ff <branch>
       [gsd-cleanup]   - Or cherry-pick:  git cherry-pick <lost-commit-sha>
       [gsd-cleanup]   - Then:          git worktree remove <worktree_path> --force && git branch -D <branch>
       ```
     - `exit $GSD_RC`(透传 gsd-sdk 退出码)。

8. **成功路径**: `OUTPUT` 含 `branch_contains_deletions` 等检测但 RC=0 时脚本正常退出 0,不打印提示(避免误报)。

9. **最终 chmod**:写完用 `chmod +x scripts/gsd-cleanup-worktree.sh`。在 commit 时用 `git update-index --chmod=+x scripts/gsd-cleanup-worktree.sh` 确保可执行位入库。

10. **不使用 heredoc 写文件**;用 Write 工具直接落盘。

**避坑**:
- 不要用 `set -e` 直接 `gsd-sdk ...`;一定要用 `||` 或 `if` 捕获,否则后续的 stderr 扫描和 HINT 输出不会执行。
- 不要试图递归删除 `node_modules` 目录——只处理 symlink。
- 路径用绝对路径,不要依赖 cwd;`REPO_ROOT` 一开始就解析。
- `find` 命令在路径不存在时不能崩,用 `2>/dev/null` 兜底。

**验证手段**:

- `./scripts/gsd-cleanup-worktree.sh /dev/null` 应进入 manifest 解析失败(空对象 `{"worktrees":[]}`)并最终 gsd-sdk 报错;不应崩;退出码非 0。
- `ls -l scripts/gsd-cleanup-worktree.sh` 确认 `-rwxr-xr-x`。
- `git diff --cached --stat` 看到脚本文件(无 staged 时用 `git status` 看 untracked)。

**目标**:在不破坏现有 `permissions.allow` 的前提下,在 `.claude/settings.json` 顶层加 `hooks` 键(若有则合并,无则新建)。

**当前状态**:文件只有 `permissions.allow` 一项。需要 `Edit` 工具:在最后的 `}`(即 `permissions` 对象的关闭)前插入逗号 + `hooks` 块。

**注入内容(精确)**:
```json
,
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Bash",
        "hooks": [
          {
            "type": "command",
            "command": "INPUT=$(cat); CMD=$(echo \"$INPUT\" | jq -r '.tool_input.command // \"\"' 2>/dev/null); if echo \"$CMD\" | grep -q 'gsd-sdk query worktree.cleanup-wave'; then echo '⚠ Use scripts/gsd-cleanup-worktree.sh instead of raw gsd-sdk call' >&2; exit 2; fi"
          }
        ]
      }
    ]
  }
```

**实现方式**:
- 用 `Edit` 工具,把原文件倒数第一行 `  }`(permissions 的关闭大括号)替换成:
  ```
    }
  ,
  "hooks": { ... }
  ```
- **或**:用 `python3 -c "import json; ..."` 重写整个文件(更安全,因为可以一次 round-trip 保证 JSON 合法)。建议用 python 重写:
  ```python
  import json, pathlib
  p = pathlib.Path('.claude/settings.json')
  data = json.loads(p.read_text())
  data['hooks'] = {
    "PreToolUse": [
      {
        "matcher": "Bash",
        "hooks": [
          {
            "type": "command",
            "command": "INPUT=$(cat); CMD=$(echo \"$INPUT\" | jq -r '.tool_input.command // \"\"' 2>/dev/null); if echo \"$CMD\" | grep -q 'gsd-sdk query worktree.cleanup-wave'; then echo '⚠ Use scripts/gsd-cleanup-worktree.sh instead of raw gsd-sdk call' >&2; exit 2; fi"
          }
        ]
      }
    ]
  }
  p.write_text(json.dumps(data, indent=2, ensure_ascii=False) + '\n')
  ```
  在 Bash 里跑这个 python3(因为 Edit/Write 工具不应被 hook 影响,Bash 工具调用本身不在 PreToolUse 的匹配字符串里,hook 会放行)。

**验证**:
- `python3 -c "import json; print(json.load(open('.claude/settings.json'))['hooks']['PreToolUse'][0]['matcher'])"` 应输出 `Bash`。
- `jq -e '.hooks.PreToolUse[0].hooks[0].command | test("gsd-sdk query worktree.cleanup-wave")' .claude/settings.json` 应输出 `true`(若有 `jq`)。
- 原有 `permissions.allow` 数组长度不变(`jq '.permissions.allow | length' .claude/settings.json` 或 python 计数)——确保没误删。

**避坑**:
- 不要用 Bash + heredoc 写文件(`cat <<EOF`),要用 Edit 工具或 python3。
- 保留顶层 2 空格缩进,与现有文件风格一致。
- 末尾要换行(很多 JSON 工具要求 trailing newline)。
- hook 字符串里 `exit 2` 是 Claude Code 的"block"语义,必须保留。
- `grep -q 'gsd-sdk query worktree.cleanup-wave'` 是子串匹配——会同时匹配 `gsd-sdk query worktree.cleanup-wave` 和 `gsd-sdk query worktree\\.cleanup-wave`(后者若有引号转义)。子串精度足够,不需要再加正则。
