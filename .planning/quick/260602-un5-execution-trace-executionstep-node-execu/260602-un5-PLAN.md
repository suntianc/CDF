# 重构工作流节点执行轨迹为时序流(execution_trace)

## Objective

把工作流节点执行从「并行 logs[] + tool_calls[]」改为**统一时序流** `execution_trace[]`,每条 step 带 `type` + `ts`,真实还原 Agent 执行轨迹:
- 把 LLM 真实输出(reasoning/summary text)抓成 `thinking` step,取代"Agent 正在思考决策..."占位
- 让 `task_start` / `task_end` / `tool_call` / `tool_result` 全部按时间戳排序,真实反映"思考 → 调工具 → 工具返回 → 再思考 → ..."的时序
- 实时 UI 和导出 JSON 都按 `step.type` 渲染,不再需要正则反推类型
- 旧的 `logs` / `tool_calls` 列保留(向后兼容历史数据),但不再写入新值

## 任务链路(5 个 atomic tasks,顺序执行)

| # | Task | 范围 | 依赖 |
|---|------|------|------|
| 1 | 类型 + DB schema | types.ts / database.ts | — |
| 2 | 后端 callback 重构 | node-executor.ts / node-executor.test.ts / workflow-runtime.ts | Task 1 |
| 3 | IPC + Store | workflow-runtime.ts / workflowStore.ts | Task 2 |
| 4 | 实时 UI 渲染 | ExecutionPanel.tsx | Task 3 |
| 5 | 导出器输出新字段 | log-exporter.ts | Task 4 |

> 注: Task 2 与 Task 3 都在 `workflow-runtime.ts` 上有改动。Task 2 改「接收 onStep + 落库」,Task 3 改「push 事件结构 + 前端 store」。为避免单文件吃太多 context,任务 2 改 runtime 的"接收"路径(替换 nodeLogsMap → nodeTraceMap + INSERT 改写 execution_trace 列),任务 3 改"推送"路径(把 node_log 事件 data.log 改为 data.step)。两步合并在同一次 commit 也可,但分开为两个 task 利于 review。

```ts
function stepToFallbackLogLine(step: ExecutionStep): string {
  switch (step.type) {
    case 'task_start':  return step.label || '[Task] 开始';
    case 'task_end':    return step.label || '[Task] 结束';
    case 'system':      return step.content || '';
    case 'thinking':    return `Agent 正在思考决策...`; // 还原老占位(老的 log-exporter 已经过滤这一行,但 logs 列回填无害)
    case 'tool_call': {
      const argStr = typeof step.args === 'string' ? step.args : JSON.stringify(step.args ?? {});
      const truncated = argStr.length > 150 ? argStr.slice(0, 150) + '...' : argStr;
      return `[工具调用] 开始执行工具: ${step.tool}，参数: ${truncated}`;
    }
    case 'tool_result':
      if (step.success) {
        const out = typeof step.output === 'string' ? step.output : JSON.stringify(step.output ?? '');
        const t = out.length > 200 ? out.slice(0, 200) + '...' : out;
        return `[工具返回] 工具 ${step.tool} 执行成功，结果: ${t}`;
      }
      return `[工具失败] 工具 ${step.tool} 执行失败，错误: ${step.error}`;
    default: return '';
  }
}
```

#### 4.3.1 辅助函数 `formatTs`

```ts
function formatTs(ts: number): string {
  const d = new Date(ts);
  const hh = String(d.getHours()).padStart(2, '0');
  const mm = String(d.getMinutes()).padStart(2, '0');
  const ss = String(d.getSeconds()).padStart(2, '0');
  const ms = String(d.getMilliseconds()).padStart(3, '0');
  return `${hh}:${mm}:${ss}.${ms}`;
}
```
