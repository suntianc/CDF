# Quick 260602-dz3 Plan 01: 移除节点输出 JSON Schema 校验机制 Summary

## 一句话总结

彻底删除上次 quick 260531-umu 引入的输出 JSON Schema 校验(2 个文件 + 3 处调用包装 + 7 个测试用例),`node-executor` 三个执行路径(Task/Loop/For-Each)回归到直接 `await invokeAgent(ctx)`,routing 提取仍从 resultText 原始输出工作。

## 改动详情

### 删除的文件
- `src/main/workflow/output-validator.ts`(executeWithValidation 包装器,带 5 轮重试 + 降级)
- `src/shared/node-output-schemas.ts`(TaskOutputSchema / ReviewOutputSchema / DegradedOutput 等 5 个 zod schema)

### 改动文件:src/main/workflow/node-executor.ts
- 删除第 20 行 `import { executeWithValidation } from './output-validator';`
- **Loop 路径(原 416-431 行)**:6 行 executeWithValidation 调用 → 1 行 `const resultText = await invokeAgent(iterationContext);`;从 `iterations.push(...)` 中删除 `structuredOutput` / `_degraded` / `_validationErrors` 两行 spread
- **For-Each 路径(原 499-516 行)**:同上模式替换;`// Validate For-Each individual items using 'task'...` 注释同步删除
- **Task/Review 路径(原 550-570 行)**:同上模式替换;`// 4. 执行 DeepAgent(... 带输出校验)` 注释改为 "单次调用"
- 保留:`extractJsonCandidate` / `extractWorkflowRouting` / `invokeAgent` 闭包 / Loop / For-Each / Task 主流程结构
- 三个 return 对象中,`structuredOutput` / `_degraded` / `_validationErrors` 三个字段全部不再产生

### 改动文件:src/main/workflow/node-executor.test.ts
- 删除 7 个 it() 块:
  - `should include structuredOutput when task node output passes validation`
  - `should retry when output fails validation and succeed on second attempt`
  - `should degrade after MAX_RETRIES (5) consecutive validation failures`
  - `should validate review node output with verdict field`
  - `should include structuredOutput in loop iteration records on validation success`
  - `should preserve routing extraction from raw output text regardless of validation`
  - `should still extract routing from raw text even when validation degrades`
- 清理 `should execute loop nodes once per iteration and pass previous output forward` 测试中 iterations 数组的 `structuredOutput` 断言(3 处)
- 保留:`extractWorkflowRouting` 单测(行 118-123)、Loop 早停测试、For-Each 5 个测试、Tool 日志回调测试、错误处理 2 个测试,共 12 个测试全部通过

1. **保留 zod 依赖**:`types.ts` / `state-schema.ts` / `tools.ts` 仍使用 zod,本计划不删 package.json 中的 zod 依赖,避免破坏面扩散

4. **不补 `output-validator.test.ts` 新文件**:该文件本来就不存在,本计划也不创建,避免无效测试套件
