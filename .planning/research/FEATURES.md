# Feature Landscape: Agent 开发工作站

**Domain:** AI Agent 开发工作站（桌面应用）
**研究日期:** 2026-05-21
**信心等级:** MEDIUM — 基于竞品公开文档和功能对比，非实时市场调研

---

## Table Stakes（必备功能）

用户期望的基础能力。缺少则产品不可用。

| 功能 | 期望原因 | 复杂度 | 说明 |
|------|----------|--------|------|
| **自然语言转代码生成** | 核心交互方式，用户通过描述需求驱动开发 | High | 需要模型理解意图并生成有效代码 |
| **多轮对话界面** | 需求澄清、迭代修改、多步骤任务沟通 | Med | Master Agent 与用户的主要交互形式 |
| **代码补全/建议** | 提升编码效率，减少重复输入 | Low | IDE 级标配功能 |
| **多文件编辑能力** | 真实开发任务涉及多个文件 | High | 创建、编辑、删除、重构跨文件 |
| **项目上下文感知** | 生成的代码需要符合项目风格、技术栈 | High | 需要 RAG 或上下文注入 |
| **LLM 供应商配置** | 用户可能已有各供应商账号，需灵活切换 | Med | OpenAI、Anthropic、本地模型等 |
| **基础文件操作** | 代码生成的执行载体 | Low | 文件读写、执行脚本 |

---

## Differentiators（差异化功能）

非必备但有竞争力的功能，可形成产品护城河。

| 功能 | 价值主张 | 复杂度 | 说明 |
|------|----------|--------|------|
| **工作流可视化编排** | 非程序员也可编排复杂 Agent 任务 | High | ReactFlow 类拖拽界面，节点并行/串行执行 |
| **MCP 支持** | 扩展 Agent 能力边界，接入外部工具生态 | Med | Model Context Protocol 是新兴标准 |
| **Agent 资产库管理** | 沉淀和复用 Agent 配置，提升效率 | Med | Agent 模板、MCP 资源包、Skills |
| **多项目管理** | 支持同时处理多个代码仓库 | Med | 项目隔离、上下文切换 |
| **任务自主执行** | 无需人工介入完成端到端任务 | High | Devin、Bolt 的核心能力 |
| **Skills 管理系统** | 封装可复用技能脚本，扩展 Agent 能力 | Med | 脚本化管理、版本控制 |
| **工作流执行监控** | 实时查看任务进度、节点状态 | Med | 节点级执行日志、错误定位 |
| **上下文窗口管理** | 长任务不丢上下文，支持会话继承 | High | 85% 阈值自动总结策略 |
| **离线优先架构** | 满足企业安全、合规需求 | Med | 本地存储，不依赖云端 |
| **主题切换** | 用户个性化体验 | Low | Shadcn UI 组件支持 |

---

## Anti-Features（刻意不做的功能）

基于项目约束和定位，明确排除。

| 功能 | 排除原因 | 替代方案 |
|------|----------|----------|
| **云端同步** | 离线优先设计，v1 单开发者使用 | 本地数据持久化 |
| **团队协作/权限管理** | v1 单用户定位 | 未来版本考虑 |
| **实时协作编辑** | 桌面应用定位，非 Web 多人协作 | — |
| **上下文压缩** | 压缩导致信息失真 | 窗口达到 85% 时总结历史 |
| **强制自动执行** | 需要用户确认的场景必须暂停 | Master Agent 询问用户 |

## 功能依赖关系

```
用户对话
    ↓
Master Agent 意图理解
    ↓
工作流编排 ──┬── Agent 节点 ── LLM 调用
    │        │
    │        └── MCP 资源
    │        │
    │        └── Skills 执行
    │
    └── 普通节点（脚本/查询）
    ↓
执行结果交付
    ↓
用户确认（如需）
```

## MVP 推荐

**优先级排序（基于 v1-MVP 需求列表）：**

1. **Agent 对话界面** — 核心交互，Master Agent 多轮对话
2. **LLM 供应商配置** — 基础依赖，其他功能都依赖模型调用
3. **Agent 资产管理** — 差异化起点，可复用配置沉淀
4. **MCP 管理** — 扩展生态接入能力
5. **Skills 管理** — 与 MCP 配合扩展能力边界
6. **工作流可视化编排** — 差异化核心，非必须但有竞争力
7. **项目管理** — 多仓库支持
8. **主题切换** — 低优先级 UI 功能

**可暂缓：**
- 多项目管理的高级隔离机制
- 工作流执行回滚/版本管理
- Agent 市场/共享功能
