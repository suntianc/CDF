# Phase 03.1: agent - Context

**Gathered:** 2026-05-26（第三版）
**Status:** Ready for planning

<domain>
## Phase Boundary

实现 Master Agent 调用子Agent 的完整流程。核心机制：
- `subagentIds` 白名单控制可委派范围
- `Agent.slug` 作为 `task(name)` 稳定调用键
- 统一 `responseFormat` 结构化输出
- Runtime 统一包装异常状态
- TaskPanel 展示委派执行状态

Phase 3.1 聚焦**子agent 调用的运行时机制**，TaskPanel 已有。

</domain>

<decisions>
## Implementation Decisions

### 1. 白名单来源（唯一）
- **D-01:** `subagentIds` 是**唯一**白名单来源，不存在其他可委派 Agent 来源
- **D-02:** 未配置 `subagentIds` 时，`task` tool **不暴露**给 Master Agent

### 2. Agent 标识体系
- **D-03:** `Agent.slug` 是 `task(name)` 的**唯一稳定键**，不得使用 UUID 或其他标识
- **D-04:** `Agent.name` 仅用于 UI 展示，不参与任何运行时逻辑

### 3. general-purpose 策略
- **D-05:** 默认 `general-purpose` subagent **始终禁用**，不接受任何形式的自动启用

### 4. Agent → deepagents subagent 映射
- **D-06:** 每个 `subagentIds` 中的 Agent 映射为 deepagents subagent：
  ```typescript
  {
    name: Agent.slug,           // task(name) 唯一稳定键
    description: Agent.description,
    systemPrompt: Agent.system_prompt,
    tools: [...],              // 必须显式传递，不继承默认工具
    model: Agent.model,
    responseFormat: DelegatedTaskResultSchema
  }
  ```
- **D-07:** **禁止默认工具继承** — `tools` 必须从 Agent 绑定的 MCP/Skills 显式构造，禁止依赖 deepagents 内置默认工具

### 5. 任务包传递
- **D-08:** 子Agent **不继承父级 checkpoint**
- **D-09:** 任务包**只通过 `task` 参数传递**（JSON 字符串），包含：
  1. 任务目标
  2. 用户约束
  3. 相关文件路径
  4. 项目根目录
  5. 期望产物
  6. 必要历史摘要

### 6. responseFormat（统一 Schema）
- **D-10:** 所有子Agent **强制使用统一 Zod schema**：
  ```typescript
  const DelegatedTaskResultSchema = z.object({
    status: z.enum(["success", "failure"]),
    artifacts: z.array(z.string()),
    summary: z.string(),
    error: z.object({
      code: z.string(),
      message: z.string()
    }).optional()
  });
  ```

### 7. 异常状态包装（Runtime 负责）
- **D-11:** Runtime 统一包装异常为 failure 格式：
  - `timeout` → `{ status: "failure", error: { code: "TIMEOUT", message: "..." } }`
  - `interrupted` → `{ status: "failure", error: { code: "INTERRUPTED", message: "..." } }`
  - `parse_failed` → `{ status: "failure", error: { code: "PARSE_FAILED", message: "..." } }`

### 8. TaskPanel 展示
- **D-12:** TaskPanel 根据 **stream metadata 和 task events** 识别 delegated task
- **D-13:** 展示状态只有三种：`running` | `success` | `failure`
- **D-14:** 失败原因通过 `error.code` 区分：`TIMEOUT` | `INTERRUPTED` | `PARSE_FAILED` | 其他

### 9. 底层机制
- **D-15:** 移除 `excludedTools: ['task']`，启用 task tool
- **D-16:** `generalPurposeSubagent: { enabled: false }` 保持不变
- **D-17:** `createDeepAgentRuntime` 增加 `subagentIds?: string[]` 参数
- **D-18:** `task(name, task)` 调用：`name` = `Agent.slug`，`task` = 任务包 JSON 字符串

</decisions>

### Agent → subagent 映射
```typescript
async function createDeepAgentRuntime(
  projectId: string,
  sessionId: string,
  currentMessage: RuntimeInputMessage,
  agentId?: string | null,
  overrides?: RuntimeModelOverrides,
  subagentIds?: string[]  // 新增
) {
  const subagents = subagentIds?.map(id => {
    const agent = db.prepare('SELECT * FROM agents WHERE id = ?').get(id);
    return {
      name: agent.slug,           // D-03: 唯一稳定键
      description: agent.description,
      systemPrompt: agent.system_prompt,
      tools: buildToolsFromMcpAndSkills(agent),  // D-07: 显式传递
      model: agent.model,
      responseFormat: DelegatedTaskResultSchema  // D-10
    };
  });

  const deepAgent = createDeepAgent({
    // ...
    subagents,  // 传给 deepagents
  });
}
```

<specifics>
## Specific Ideas

- `Agent.slug` 作为 `task(name)` 唯一稳定键，确保跨会话可追溯
- `tools` 从 MCP/Skills 显式构造，禁止默认继承
- Runtime 捕获异常后主动包装为符合 schema 的 failure 响应
- TaskPanel 状态简化：只展示 running/success/failure，细节从 error.code 读取

</specifics>
