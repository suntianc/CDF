### Gaps Summary

**2 tests fail** in `src/main/deepagent/runtime.test.ts`:

1. **"should pass subagents to createDeepAgent when subagentIds provided"** (line 256)
   - Root cause: UUID validation in runtime.ts (line 380-385) rejects 'agent-1' as invalid UUID format
   - Evidence: `console.warn([runtime] Invalid UUID format for subagentId: agent-1)`
   - Analysis: Production agent IDs use `crypto.randomUUID()` (valid UUIDs). The test uses simplified IDs ('agent-1', 'agent-2') that don't match UUID v4 format. The validation is correct for production — it prevents accidental misuse. This is a test data quality issue.

2. **"should use generated slug when agent.slug is null"** (line 318)
   - Root cause: Same UUID validation issue — subagents list is empty due to 'agent-2' being rejected

**Impact assessment**: The implementation is correct for production use (UUID validation is appropriate safeguard). Tests need valid UUID format IDs or a mock setup that bypasses UUID validation.

**Suggested fix for tests**: Replace 'agent-1'/'agent-2' with valid UUIDs in test mock setup, OR add a test mode that skips UUID validation.
