## 错误处理（审查新增）

| Error Case | MVP 处理 |
|-----------|---------|
| 子Agent 失败状态 | status: "failure"，TaskPanel 显示 error message |
| 超时保护 | status: "timeout"，可配置阈值（默认 5 分钟） |
| JSON 解析失败 | status: "parse_failed"，作为普通消息处理 |
| 用户取消/中断 | status: "interrupted"，状态回写到 checkpoint |

**Notes:** 基础错误处理是 MVP 必须，不是 deferred

---

## 审查意见（来源：Suntc 君）

**5 个关键整改点：**
1. 不要假设 task tool 入参是 `agent_id/context`，正确格式是 `task(name, task)`
2. 结构化 JSON 不要只靠 system prompt 约束，用 `responseFormat` / Zod schema
3. 谨慎启用 `general-purpose` subagent，显式覆盖或关闭
4. "最小上下文"需要 7 个具体要素
5. 错误处理不能完全 deferred，MVP 至少要有基础错误处理
