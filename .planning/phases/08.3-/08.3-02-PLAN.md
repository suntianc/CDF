    **3. `getRelativePath(absolutePath: string, projectRoot: string): string`**:
    - Import: `import path from 'path';` (Node `path` module — available in renderer through electron-vite's polyfill or `node:` protocol alias; verify with `node -e "require('path')"` returning successfully)
    - Body: `return path.relative(projectRoot, absolutePath).replace(/\\/g, '/').replace(/^\.\//, '').replace(/^\//, '');`
    - The 3-step normalize: (1) compute relative, (2) convert backslashes to forward slashes (Pitfall #5), (3) strip leading `./` and `/` (C-04)
    - Return type: `string`. Always returns a POSIX path with no leading `./` or `/`.

    **4. `parseAtTokens(text: string): Array<{ path: string; kind: 'file' | 'dir'; start: number; end: number }>`**:
    - Body: regex scan with `const regex = /(?<=^|\s)@([\w./-]+)/g;` — lookbehind for `@` at start-of-string or after whitespace (avoids matching `user@host.com` style emails)
    - The captured group `[\w./-]+` allows word chars, dots, slashes, hyphens
    - For each match: extract the captured path, infer `kind = path.endsWith('/') ? 'dir' : 'file'`, and return `{ path, kind, start: match.index + 1, end: match.index + 1 + path.length }` (the `+1` accounts for the `@` prefix; `start` points AT the `@` character, `end` points past the last path char)
    - Note: this regex does NOT match `@` followed by whitespace (which would be an empty path — the user just typed `@` and the popup is still open)
    - Note: this regex does NOT match `@some/path with space.ts` — stops at the first non-`[\w./-]` char
    - Return type: array (may be empty); never null/undefined

    **5. `isPathIgnored(relativePath: string, ignoreRules: string): boolean`**:
    - Lightweight renderer-side check: the renderer does not need the full `ignore@5` instance, just a simple "is this path in the ignore rules?" check for the F-02 tooltip
    - Imports: `import ignore from 'ignore';` (the renderer CAN import `ignore@5` since it's pure JS — verify with `node -e "require('ignore')"` returning successfully in the renderer build)
    - Body: `const ig = ignore(); ig.add(ignoreRules); return ig.ignores(relativePath);`
    - `ignoreRules` is the raw `.gitignore` file CONTENT (a string), not a parsed instance. The caller (Plan 03) gets this from main once on popup open and caches it.
    - Return type: `boolean`. Returns `false` when `ignoreRules` is empty or `relativePath` is empty (defensive).
