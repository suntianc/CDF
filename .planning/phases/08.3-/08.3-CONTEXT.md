# Phase 08.3: 增加@引用文件功能 - Context

**Gathered:** 2026-06-07
**Status:** Ready for planning

<domain>
## Phase Boundary

在 CDF 聊天输入框中支持 `@` 引用项目内的文件/目录，让用户在自然语言消息中轻松把本地资源"指向"给 Master Agent。

交付内容：
- 触发与选择：用户在 composer/welcome 输入框敲入 `@`，弹出文件/目录搜索 popup（沿用 slash popup 模式：cmdk + Radix Popover）。
- 搜索范围：当前 projectPath 根目录下的文件与目录，排除 `.gitignore` 命中的路径。
- token 渲染：选中后插入 `@relative/path` 字面量，**视觉**上以统一 token 呈现（文件 = file 图标，目录 = folder 图标），与 Phase 08.1 slash token 一致的 literal-string 契约。
- 消息发送：`@` 引用**不**展开文件内容、不附加工具指令，原样保留 `@relative/path` 在用户消息中。
- 消息历史：发送后的历史消息继续渲染为统一 token，与输入框一致。
- 失败反馈：因为范围严格按 `.gitignore` 过滤，第一版几乎无失败；仅在用户手动键入一个被 `.gitignore` 排除的路径时给出轻量提示。

不在范围：
- ❌ 跨项目引用、多 projectWorkspace 跨项目搜索
- ❌ LLM 收到路径后自动读取内容、附件化、文件内容注入
- ❌ 任意系统级路径访问（仅当前 projectPath 内）
- ❌ 远程 git、HTTP 文件内容抓取
- ❌ 编辑/重写 token 文本、双击进入编辑等高级交互
- ❌ 重发/编辑消息时的 token 重建 UI（仅准备解析工具）

</domain>

<decisions>
## Implementation Decisions

### A. 触发与 popup

- **A-01:** `@` 触发：在 ChatArea 输入框光标位置检测到 `@` 且不在已有 word 内部时，弹出文件搜索 popup。
- **A-02:** 沿用 slash popup 框架：cmdk `Command.List` + Radix `Popover`，复用 Phase 5/6 已稳定的 `SlashCommandPopup` 模式。
- **A-03:** popup 顶部加一个过滤输入（自动聚焦到当前 `@` 之后的查询字符串），支持模糊匹配项目内路径（按相对路径/文件名匹配，UTF-8 NFKC + 变体选择器去除，与 Phase 8 D-05/D-06 一致）。
- **A-04:** 键盘导航沿用 slash popup：`↑/↓` 选择、`Enter` 选中插入、`Esc` 关闭、`Tab` 关闭（不插入）；与 Phase 6 D-07 行为对齐。
- **A-05:** 选中后行为：把 `@` 到当前光标的查询片段整段替换为 `@relative/path` 字符串（含末尾空格 + 闪烁光标沿用 slash token 模式）。

### B. 搜索范围与 .gitignore 过滤

- **B-01:** 范围 = 当前 active project 的 projectPath 根目录（通过 projectStore 获取）。不存在的 projectPath 不弹 popup。
- **B-02:** 解析项目根 `.gitignore`（使用 `ignore@5` 或自写 `picomatch` + 行注释剥离），命中模式不显示。
- **B-03:** 默认额外排除系统级生成/依赖目录（即使未写进 `.gitignore`）：`node_modules`、`out`、`dist`、`.git`、`.next`、`.cache`。
- **B-04:** 同时列出文件与目录；目录以 folder 图标显示，文件以 file 图标显示。
- **B-05:** 第一版采用浅层枚举（递归深度上限 6 层 + 总数上限 5000 条），避免超大仓库卡顿；后续如需 glob/fzf 风格可迭代。
- **B-06:** 用户手动键入 `@relative/path` 时，发送前在 renderer 侧用同一份 `.gitignore` 规则做校验：被排除的路径在 popup 显示一条轻提示（`该路径在 .gitignore 中被排除`），其他情况不阻断。

### C. token 渲染（输入框 + 历史消息）

- **C-01:** 新组件 `src/renderer/src/components/AtMention/AtToken.tsx`，导出 `<AtToken path={...} kind="file" | "dir" />`。
- **C-02:** 输入框 overlay（沿用 Phase 08.1 overlay 模式）：扫描 `inputVal`，把 `@relative/path` 片段渲染成 `<AtToken>`，其他文本保持透明 textarea 透出。
- **C-03:** 历史消息渲染：`MessageItem` 同样把 `@relative/path` 渲染为 `<AtToken>`，保持视觉一致。
- **C-04:** 路径前缀统一为 `relative/path`（无前导 `./`，无前导 `/`），避免歧义。`getRelativePath(absolutePath, projectPath)` 工具集中放在 `src/renderer/src/lib/commands/pathUtils.ts`（新建）。
- **C-05:** Backspace 原子删除：与 Phase 08.1 D-04 相同的事件拦截策略——光标位于 token 末尾空格时按 Backspace 一次性删除整个 `@relative/path `。
- **C-06:** token 内部 `onMouseDown={(e) => e.preventDefault()}` 保持 textarea 焦点不丢。

### D. 消息发送

- **D-01:** `useSessionStore.sendMessage` 入参不需新增字段；用户消息原文（含 `@relative/path`）照旧发送。
- **D-02:** 不展开文件内容到 payload，不增加 `attachments` / `context` 字段。
- **D-03:** 路径格式发送：原样保留 `@relative/path`；如用户删除 token 仅剩纯文本则退化正常发送。

### E. IPC / Main process

- **E-01:** 新增 IPC `project:listAtMentionCandidates(rootPath, query)`：在 main 端跑 `.gitignore` 解析 + 浅层枚举，返回 `{ path: relativePath; kind: 'file' | 'dir' }[]`。
- **E-02:** Renderer popup 打开时调用 IPC 拉候选；popup 关闭后释放缓存（避免大仓库内存滞留）。
- **E-03:** IPC 返回数量上限 5000；超过时附 `truncated: true` 字段，由 popup 给出顶部提示。
- **E-04:** 解析 `.gitignore` 使用 `ignore@5` 库（与 chokidar 路径处理一致；新增 devDep）。
- **E-05:** `preload/index.ts` 暴露 `window.electronAPI.project.listAtMentionCandidates`，TypeScript 类型补全。

### F. 失败/反馈策略

- **F-01:** 既然范围已经按 `.gitignore` 严格过滤，正常 UI 不会进入"失败"状态。
- **F-02:** 用户手动键入 `@some/ignored/path` 时，渲染层在 token 后显示一个 tooltip/小标签：「该路径在 .gitignore 中被排除，发送后 LLM 可能读不到」，但不阻止发送。
- **F-03:** popup 顶部空结果行显示「未找到匹配文件」并提供 Sonner 静默（不弹 toast，避免打扰）。

### G. 测试范围

- **G-01:** 必须覆盖：popup 过滤、选择插入 token、`.gitignore` 排除生效、目录路径保留、消息历史 token 渲染、`@path` 解析为 `{ path, kind }`、手工输入被忽略路径的 tooltip。
- **G-02:** 单元测试 + 关键集成测试（dispatcher 仍走非 slash 路径，回归原有 slash 行为）。
- **G-03:** 暂不写 E2E（与 Phase 08.1 一致）。

### Claude's Discretion
- 浅层枚举的递归/数量上限参数细节（C-01/C-02 中如阈值、目录遍历顺序等可由实现者微调）。
- `<AtToken>` 的具体视觉尺寸、圆角、字号、颜色（沿用 Phase 08.1 SlashToken 风格即可）。
- 模糊匹配排序策略（路径前缀匹配 > 名称包含 > 模糊打分，由实现者决定）。

</decisions>

<specifics>
## Specific Ideas

- 用户场景：用户在 composer 写「请帮我修一下 @src/renderer/src/lib/commands/dispatcher.ts 里 planOnly 那段逻辑」，按下 Enter 后 LLM 看到的就是带 `@path` 的原话，由 agent 自己用读文件工具处理。
- 视觉上 token 与 Phase 08.1 slash token 保持一致（中性背景 + 圆角 + 小图标 + Title Case 风格化路径），形成统一的「chip 语言」。
- `.gitignore` 默认排除加上 `node_modules` 等是符合开源社区惯例的安全选择，第一版直接默认加上不暴露开关。
- 目录引用 token 末尾带 `/` 视觉提示（例如 `@src/components/🗂/`），方便用户区分。
- 模糊匹配沿用 cmdk 默认打分（无需自写 fuzzy）。

</specifics>
