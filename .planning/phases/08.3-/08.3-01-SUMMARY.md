## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Test fixture writeFileSync before mkdirSync**
- **Found during:** Task 2 — `listCandidates lists files and directories with kind suffix`
- **Issue:** Test created `src/index.ts` via `writeFileSync` before `mkdirSync('src')`, causing `ENOENT`
- **Fix:** Reordered the test to mkdir first, then writeFile
- **Files modified:** `src/main/at-mention/candidate-lister.test.ts`
- **Commit:** `d5e25aa`

**2. [Rule 1 - Bug] .gitignore test pattern missing `build/`**
- **Found during:** Task 2 — `excludes paths matched by project .gitignore (B-02)`
- **Issue:** Test .gitignore contained only `*.log` but expected `build/` to be excluded; the test was written assuming both patterns were in the file
- **Fix:** Updated test .gitignore to `*.log\nbuild/\n` to match the assertion
- **Files modified:** `src/main/at-mention/candidate-lister.test.ts`
- **Commit:** `d5e25aa`

**3. [Rule 1 - Bug] toPosix test assumed backslashes on Unix**
- **Found during:** Task 1 — `toPosix converts platform-native separators to POSIX style`
- **Issue:** Test hard-coded `foo\\bar\\baz` and expected them to be replaced; on macOS `path.sep === '/'` so backslashes pass through unchanged
- **Fix:** Rewrote test to be platform-aware: input is `path.sep`-joined, expected output is always POSIX
- **Files modified:** `src/main/at-mention/gitignore-loader.test.ts`
- **Commit:** `47b4a36`

**4. [Rule 1 - Bug] MAX_DEPTH=6 required directory cap, not just BFS depth check**
- **Found during:** Task 2 — `enforces MAX_DEPTH = 6 (B-05)`
- **Issue:** Plan spec said depths 1-6 appear but depth 7+ does not. Initial implementation listed child directories at depth 7 (because the BFS check `depth < MAX_DEPTH` only prevented enqueue, not listing). Test expected depth-7 directories to be hidden.
- **Fix:** Added a `depth === MAX_DEPTH` short-circuit: at depth=6 we list files but skip directory entries (do not list or enqueue them)
- **Files modified:** `src/main/at-mention/candidate-lister.ts`
- **Commit:** `d5e25aa`
