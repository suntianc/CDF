<objective>
Ship the main-process infrastructure for the `@` file mention feature: `.gitignore` parser, shallow BFS candidate enumerator, IPC channel `project:listAtMentionCandidates`, preload bridge, and shared types. This unblocks Plan 02 (renderer popup) and Plan 03 (ChatArea wiring) by giving them a typed, contract-stable API to call.

Purpose: The renderer's `<AtMentionPopup>` needs a stable, minimal IPC contract to fetch candidates. The renderer cannot do `fs.readdir` (contextBridge forbids it), so the main process must own all FS IO. This plan isolates that work in 3 small modules (loader / enumerator / handler) so the renderer can consume them through one typed bridge method.

Output: 3 new main-process modules + 3 new test files + 3 modified integration points (ipc-handlers.ts, preload/index.ts, shared/types.ts) + 1 new devDep (`ignore@5.3.2`).
</objective>

    Create `src/main/at-mention/candidate-lister.ts`. Single file with one exported function and one exported interface:

    ```typescript
    export interface ListResult {
      candidates: string[];   // POSIX 相对路径；目录以 '/' 结尾
      truncated: boolean;
    }
    export function listCandidates(projectRoot: string): ListResult
    ```

    Constants at module scope (NOT in the function body — reused across calls):
    - `const MAX_DEPTH = 6;` (B-05)
    - `const MAX_COUNT = 5000;` (B-05/E-03)

    `listCandidates(projectRoot)` body:
    1. `const ig = loadGitignore(projectRoot);` — calls the loader from Task 1.
    2. Initialize `const result: string[] = []; let truncated = false;`.
    3. Build a BFS queue: `const queue: Array<{ dir: string; depth: number }> = [{ dir: projectRoot, depth: 0 }];`
    4. While-loop: `while (queue.length > 0) { const { dir, depth } = queue.shift()!; if (depth > MAX_DEPTH) continue; ... }` — the `!` is safe because the while condition guarantees non-empty.
    5. Inside the loop, `let entries; try { entries = fs.readdirSync(dir, { withFileTypes: true }); } catch (err) { continue; }` — silently skip dirs we can't read (perms, broken symlinks).
    6. For each `entry` in `entries`:
       - First check `if (result.length >= MAX_COUNT) { truncated = true; break; }` — outer-loop break (use a labeled break or a `done` flag to exit both the `for` and the `while`).
       - Compute `const abs = path.join(dir, entry.name);`
       - Compute `const relPosix = toPosix(path.relative(projectRoot, abs));`
       - Compute `const ignorePath = entry.isDirectory() ? relPosix + '/' : relPosix;` — directory paths need a trailing `/` for `ig.ignores()` to match (ignore@5 treats `build` and `build/` differently).
       - `if (ig.ignores(ignorePath)) continue;` — skip ignored paths (B-02/B-03).
       - `result.push(entry.isDirectory() ? relPosix + '/' : relPosix);` — directories are pushed WITH trailing `/`; files WITHOUT.
       - If `entry.isDirectory()` AND `depth < MAX_DEPTH`: `queue.push({ dir: abs, depth: depth + 1 });` — only recurse into non-ignored directories within depth limit.
    7. After the BFS, `return { candidates: result.sort(), truncated };` — sort alphabetically for deterministic popup order (cmdk fuzzy will re-sort interactively).
