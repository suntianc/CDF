# Wave 2: 工作流执行引擎 — LangGraph.js 集成

## Objective

实现独立的工作流执行引擎：基于 LangGraph.js StateGraph，将 ReactFlow 图定义转换为可执行的图，每个 Agent 节点通过 DeepAgent 实例执行（继承 LLM/MCP/Skills）。支持条件分支、循环保护、并行 fan-out/fan-in，执行状态通过 IPC 事件流实时推送。关键约束：独立 workflow runtime，不复用 chat runtime（D-16a）。

</read_first>

<action>
创建 `src/main/workflow/node-executor.ts`：

1. 导出 `createAgentNodeExecutor(node: WorkflowNode)` 工厂函数：
   - 返回 `(state: WorkflowStateType) => Promise<Record<string, unknown>>`
   - 从数据库加载 Agent 定义：`db.prepare('SELECT * FROM agents WHERE id = ?').get(node.data.agentId)`
   - 加载 Agent 关联的 MCP servers：JOIN agent_mcp_servers + mcp_servers
   - 加载 Agent 关联的 Skills：SELECT FROM agent_skills
   - **创建 DeepAgent 实例**：复用 `src/main/deepagent/runtime.ts` 中的构建 helper（createLangChainModel、MCP adapter 初始化、Skills 加载），但不复用 chat runtime 的 session/checkpoint 逻辑（D-16a）
   - 构造 user message：只传递 `state.inputs` 和相关上游 `state.nodeOutputs`（通过 createNodeStateExtractor 提取），不传递整个 state
   - 通过 DeepAgent 实例执行（支持 tool calling、MCP 工具调用、Skills 执行），而非裸 LLM 调用
   - 收集 Agent 执行结果（包括 tool call 历史）
   - 返回 `{ result: agentOutput, toolCalls: [...], nodeId: node.id }`

2. 导出 `createNodeStateExtractor(nodeId: string, upstreamNodeIds: string[])` 辅助函数：
   - 从 state 中提取当前节点需要的切片：inputs + 指定上游节点的 nodeOutputs
   - 防止上下文窗口溢出（D-16b）

3. 错误处理：
   - Agent 定义不存在 → 抛出 `AgentNotFoundError`
   - DeepAgent 执行超时 → 抛出 `AgentTimeoutError`
   - 所有异常被 graph-builder 的 try/catch 捕获，写入 state.errors
</action>

- .planning/phases/04-workflow-system/04-CONTEXT.md D-09/D-10/D-13/D-14/D-15 (条件分支、循环路由、LangGraph 驱动)
</read_first>

<action>
创建 `src/main/workflow/graph-builder.ts`，包含两部分：

**Part A — 条件路由 + 循环保护**：
1. 导出 `createConditionalRouter(condition: string, maxIterations?: number)` 函数：
   - 返回路由函数 `(state) => string`
   - 循环保护：检查 `state.routing[\`__loop_${condition}\`]` 计数器，>= maxIterations 时返回 `'__max_iterations_exceeded__'`
   - 从 `state.routing[condition]` 读取路由决策
2. 导出 `MAX_LOOP_ITERATIONS = 10` 常量（全局硬限制）

**Part B — ReactFlow → StateGraph 转换**：
1. 导出 `buildWorkflowGraph(workflowDef, nodeExecutor)` 函数：
   - 创建 `new StateGraph(WorkflowState)` builder
   - 遍历 nodes：跳过 start/end，对 agent 节点 `builder.addNode(node.id, async (state) => { try {...} catch {...} }, { retryPolicy })`
   - 节点成功：返回 `{ nodeOutputs: { [node.id]: result } }`
   - 节点失败：返回 `{ errors: [{ nodeId, error, timestamp }], routing: { [\`${node.id}_status\`]: 'failed' } }`
   - 遍历 edges：source=start→START, target=end→END, 有 condition→addConditionalEdges, 普通→addEdge
   - 返回 builder（不 compile）
</action>

<acceptance_criteria>
- 文件 `src/main/workflow/graph-builder.ts` 存在
- 包含 `export function createConditionalRouter`
- 路由函数包含 maxIterations 循环保护检查
- 包含 `export const MAX_LOOP_ITERATIONS = 10`
- 包含 `export function buildWorkflowGraph`
- 使用 `new StateGraph(WorkflowState)`
- 跳过 type=start/end 节点
- 条件边使用 `builder.addConditionalEdges`
- 节点异常写入 state.errors
</acceptance_criteria>

<action>
**Part A — 工作流运行时**：创建 `src/main/workflow/workflow-runtime.ts`

1. 导出 `runWorkflow(params: { workflowId, projectId, triggerSource, input?, onEvent? })` 函数：
   - executionId = `crypto.randomUUID()`
   - 从 DB 加载 workflow 定义，插入 workflow_executions 记录（status='running'）
   - 创建 SqliteSaver checkpointer，thread_id = `workflow-${executionId}`（独立 namespace，不与 chat 共享）

   - 记录 workflow_node_runs（node_start/end/error 时更新）

2. 导出 `stopWorkflow(executionId)` — 标记为 'stopped'

3. 注册 IPC handler：`workflow:run` → runWorkflow, `workflow:stop` → stopWorkflow
   - 使用 `BrowserWindow.getAllWindows()[0].webContents.send(channel, data)` 推送事件
