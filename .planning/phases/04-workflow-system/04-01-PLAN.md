<action>
在 `src/shared/types.ts` 末尾追加以下类型定义：

1. `WorkflowNodeType` 类型：`'start' | 'agent' | 'end'`（D-05/D-08a：v1 节点类型固定）
2. `WorkflowNode` interface：`{ id: string; type: WorkflowNodeType; position: { x: number; y: number }; data: { label: string; agentId?: string; description?: string; retryCount?: number; failureStrategy?: 'retry' | 'skip' | 'stop' } }`
3. `WorkflowEdge` interface：`{ id: string; source: string; target: string; sourceHandle?: string; targetHandle?: string; metadata?: { condition?: string; maxIterations?: number; targets?: Record<string, string> } }`（D-10 循环通过边 metadata 表达）
4. `WorkflowDefinition` interface：`{ nodes: WorkflowNode[]; edges: WorkflowEdge[]; viewport?: { x: number; y: number; zoom: number } }`
5. `Workflow` interface：`{ id: string; project_id: string; name: string; description?: string; graph_data: WorkflowDefinition; status: 'draft' | 'active'; created_at: number; updated_at: number }`
6. `WorkflowExecutionStatus` 类型：`'pending' | 'running' | 'completed' | 'failed' | 'stopped'`
7. `WorkflowExecution` interface：`{ id: string; workflow_id: string; project_id: string; trigger_source: 'editor' | 'chat' | 'schedule'; status: WorkflowExecutionStatus; input: Record<string, unknown>; output?: Record<string, unknown>; error?: string; started_at: number; ended_at?: number }`（D-16d triggerSource 区分）
8. `WorkflowNodeRunStatus` 类型：`'pending' | 'running' | 'completed' | 'failed' | 'skipped'`
9. `WorkflowNodeRun` interface：`{ id: string; execution_id: string; node_id: string; node_name: string; status: WorkflowNodeRunStatus; input?: Record<string, unknown>; output?: Record<string, unknown>; error?: string; error_type?: string; retry_count: number; started_at: number; ended_at?: number }`
10. `WorkflowStreamEvent` 类型（D-17 事件流）：
    ```
    | { type: 'workflow_start'; executionId: string; workflowId: string }
    | { type: 'node_start'; executionId: string; nodeId: string; nodeName: string }
    | { type: 'node_end'; executionId: string; nodeId: string; duration_ms: number; outputKeys: string[] }
    | { type: 'node_error'; executionId: string; nodeId: string; errorType: string; errorMessage: string; retryCount: number }
    | { type: 'workflow_end'; executionId: string; status: WorkflowExecutionStatus; duration_ms: number }
    | { type: 'loop_terminated'; executionId: string; edgeId: string; iterationCount: number }
    ```
11. 在 `ElectronAPI.db` 接口中追加工作流方法签名（参照现有 getAgents/saveAgent/deleteAgent 模式）：

    - `getWorkflowExecutions: (workflowId: string) => Promise<WorkflowExecution[]>`
    - `getWorkflowExecution: (id: string) => Promise<WorkflowExecution | undefined>`
    - `getWorkflowNodeRuns: (executionId: string) => Promise<WorkflowNodeRun[]>`
12. 在 `ElectronAPI` 中新增 `workflow` 命名空间：
    - `runWorkflow: (workflowId: string, projectId: string, triggerSource: string, input?: Record<string, unknown>) => Promise<string>` (返回 executionId)

<action>
**Part A — IPC handlers**：在 `src/main/ipc-handlers.ts` 的 `registerIpcHandlers()` 函数末尾追加：

1. `db:getWorkflows` — `db.prepare('SELECT * FROM workflows WHERE project_id = ? ORDER BY updated_at DESC').all(projectId)`，graph_data 从 JSON parse 为对象
2. `db:getWorkflow` — `db.prepare('SELECT * FROM workflows WHERE id = ?').get(id)`，graph_data parse
3. `db:saveWorkflow` — INSERT OR REPLACE 逻辑：如果 id 存在则 UPDATE，否则 INSERT。graph_data JSON.stringify 存储。updated_at = Date.now()
4. `db:deleteWorkflow` — `db.prepare('DELETE FROM workflows WHERE id = ?').run(id)`
5. `db:getWorkflowExecutions` — `db.prepare('SELECT * FROM workflow_executions WHERE workflow_id = ? ORDER BY started_at DESC').all(workflowId)`，input/output parse
6. `db:getWorkflowExecution` — `db.prepare('SELECT * FROM workflow_executions WHERE id = ?').get(id)`，input/output parse
7. `db:getWorkflowNodeRuns` — `db.prepare('SELECT * FROM workflow_node_runs WHERE execution_id = ? ORDER BY started_at').all(executionId)`，input/output parse

所有 handler 使用 `ipcMain.handle(channel, async (event, ...args) => {...})` 模式。
graph_data、input、output 字段存储为 JSON string，读取时 parse。

1. 在 `db` 命名空间末尾追加 CRUD 方法（channel 名称必须与 Part A 完全一致）：
   - `getWorkflows: (projectId) => ipcRenderer.invoke('db:getWorkflows', projectId)`
   - `getWorkflow: (id) => ipcRenderer.invoke('db:getWorkflow', id)`
   - `saveWorkflow: (workflow) => ipcRenderer.invoke('db:saveWorkflow', workflow)`
   - `deleteWorkflow: (id) => ipcRenderer.invoke('db:deleteWorkflow', id)`
   - `getWorkflowExecutions: (workflowId) => ipcRenderer.invoke('db:getWorkflowExecutions', workflowId)`
   - `getWorkflowExecution: (id) => ipcRenderer.invoke('db:getWorkflowExecution', id)`
   - `getWorkflowNodeRuns: (executionId) => ipcRenderer.invoke('db:getWorkflowNodeRuns', executionId)`

2. 新增 `workflow` 命名空间：
   - `runWorkflow: (workflowId, projectId, triggerSource, input?) => ipcRenderer.invoke('workflow:run', workflowId, projectId, triggerSource, input)`
   - `stopWorkflow: (executionId) => ipcRenderer.invoke('workflow:stop', executionId)`
   - `onWorkflowEvent: (executionId, callback) => { const channel = \`workflow:event-${executionId}\`; const listener = (event, data) => callback(event, data); ipcRenderer.on(channel, listener); return () => { ipcRenderer.removeListener(channel, listener); }; }`

<action>
创建 `src/renderer/src/stores/workflowStore.ts`，使用 Zustand：

```
interface WorkflowState {
  workflows: Workflow[];
  currentWorkflow: Workflow | null;
  executions: WorkflowExecution[];
  currentExecution: WorkflowExecution | null;
  nodeRuns: WorkflowNodeRun[];
  isLoading: boolean;
  error: string | null;

  fetchWorkflows: (projectId: string) => Promise<void>;
  fetchWorkflow: (id: string) => Promise<void>;
  saveWorkflow: (workflow: any) => Promise<Workflow>;
  deleteWorkflow: (id: string) => Promise<void>;
  setCurrentWorkflow: (workflow: Workflow | null) => void;

  fetchExecutions: (workflowId: string) => Promise<void>;
  fetchNodeRuns: (executionId: string) => Promise<void>;
  runWorkflow: (workflowId: string, projectId: string, triggerSource: string, input?: Record<string, unknown>) => Promise<string>;
  stopWorkflow: (executionId: string) => Promise<void>;
  subscribeToExecution: (executionId: string) => () => void;
}
```

关键实现：
- `fetchWorkflows` 调用 `window.electronAPI.db.getWorkflows(projectId)`
- `saveWorkflow` 调用 `window.electronAPI.db.saveWorkflow(workflow)`，成功后刷新列表
- `runWorkflow` 调用 `window.electronAPI.workflow.runWorkflow(...)`，返回 executionId
- `subscribeToExecution` 调用 `window.electronAPI.workflow.onWorkflowExecution(executionId, callback)`，callback 更新 currentExecution/nodeRuns 状态
- 错误处理：set error state, re-throw
- Export: `export const useWorkflowStore = create<WorkflowState>(...)`
</action>
