# Phase 08.1: 斜杠命令输入框内联渲染 — Specification

**Created:** 2026-06-05
**Ambiguity score:** 0.15 (gate: ≤ 0.20)
**Requirements:** 8 locked

## Goal

When the user selects a slash command in the Master Agent composer (or welcome screen), the plain text `/cmd-name` is replaced with an inline **styled token** — a rounded pill containing the source-specific icon and Title-Cased command name — followed by a text cursor for typing arguments; the underlying `inputVal` remains the literal `/cmd-name args` string so the dispatcher contract is unchanged.

## Background

**Current state (Phase 8.1 starting point):**

- `ChatArea.tsx:841` (welcome) and `:1127` (composer) wrap a plain `<textarea>` with a Radix `<Popover>` containing `<SlashCommandPopup>`.
- On Tab/Enter, `handleSlashSelect` (`:697`) / `handleSlashInsert` (`:716`) call `setInputVal(cmd + ' ')` — the command is inserted as **plain text**.
- `dispatcher.resolve(inputVal, commands)` expects a string starting with `/cmd-name`. PluginRewrite rewrites the literal to `"请调用 X 工具，参数：args"` before sending to the LLM (dispatcher.ts:61–69) — the LLM never sees `/cmd-name`.
- 7 source types (`system`, `mcp`, `skill:project`, `skill:global`, `workflow`, `cmd:project`, `cmd:system`) have a 7-color badge palette (`SOURCE_TEXT_COLOR` in SlashCommandPopup.tsx:24) but no per-source icon mapping at the input layer.
- History messages are stored as plain text in `messages.content` (no metadata field).
- IME composition is protected via `isComposingRef` + 200ms timer (ChatArea.tsx:150–152, 648–680).

**Gap:** Plain text in the input gives no visual cue that the slash command will be expanded into a prompt by the dispatcher. The reference image shows a rounded pink pill with a cube icon and "Gsd Fast" — the user wants this UI affordance to clarify "this command is a prompt shortcut."

**Reference:** Claude Code's `SKILL.md` design (skills are prompt-based — `/skill-name` expands to the file body via `$ARGUMENTS` substitution). The token in CDF is a *visual compression* of "this slash command will expand to a longer prompt."

## Requirements

### 1. Token rendering on slash command selection

- **Current:** Tab/Enter on popup calls `setInputVal(cmd + ' ')` (ChatArea.tsx:705, 717) — plain text only.
- **Target:** After Tab/Enter, the input visually displays a styled token (rounded pill, source-specific icon, Title-Cased name) in place of the leading `/cmd-name`; the text cursor `|` sits immediately after the token's trailing space; the underlying state still holds the literal `/cmd-name args` string.
- **Acceptance:** DOM query for `data-testid="slash-token"` returns the token element after selection; `inputVal` (controlled state) equals `/cmd-name ` (with trailing space); popup is closed.

### 2. Per-source icon mapping

- **Current:** 7 source types share rendering by source-label text only (badge in popup); no icon is shown at the input layer.
- **Target:** Each source has a dedicated lucide icon, e.g., `system` → `Sparkles`, `mcp` → `Wrench`, `skill:project` → `GraduationCap`, `skill:global` → `BookOpen`, `workflow` → `Play`, `cmd:project` → `FileText`, `cmd:system` → `FileCog`. Icon lives inside the token pill to the left of the command name.
- **Acceptance:** `getSlashTokenIcon(source)` returns distinct components for all 7 sources; visual snapshot test confirms 7 distinct icons render.

### 3. Title-Case name display

- **Current:** Command name rendered as-is (e.g., `gsd-fast` shown as `gsd-fast` in the popup; no token exists at input).
- **Target:** Token label is Title-Cased: first letter of every hyphen-separated segment is uppercased, hyphens preserved — `gsd-fast` → `Gsd-Fast`, `goal` → `Goal`, `code-review` → `Code-Review`.
- **Acceptance:** Unit test on `formatTokenLabel(name)` returns Title-Cased string for representative inputs (`gsd-fast`, `goal`, `code-review`, `run-skill-generator`).

### 4. Atomic Backspace (whole-token deletion)

- **Current:** Backspace deletes one character at a time (standard textarea behavior).
- **Target:** When the caret is immediately after a token (i.e., the token occupies the start of `inputVal` and the caret sits at position `len('/cmd-name ')`), a single Backpress deletes the entire `/cmd-name ` prefix (token + trailing space) — not character-by-character.
- **Acceptance:** With `inputVal = '/goal fix login'` (token `/goal` + space + args), pressing Backspace once from end → `inputVal` becomes `''` (token + space removed in one keystroke). Test asserts `inputVal.length === 0` after one Backspace.

### 5. Click inside token is a no-op

- **Current:** N/A (no token exists).
- **Target:** Clicking anywhere inside a token (head, middle, or tail) does NOT place the caret inside the token; the user cannot edit a single character of the command name. To change the command, the user must Backspace the whole token and re-select.
- **Acceptance:** Synthetic click event dispatched at the geometric center of a rendered token → `selectionStart` remains at its pre-click position (either end-of-input or some non-token position), and `inputVal` is unchanged.

### 6. Single token, start-of-input only (preserves v1.1 D-12)

- **Current:** `handleSend` checks `inputVal.startsWith('/')` to treat input as a slash command (ChatArea.tsx:625). Mid-text slashes (e.g., `请运行 /gsd-fast`) are passed through as plain text.
- **Target:** After a token is rendered at the start, pressing `/` again does NOT re-open the SlashCommandPopup (a second token is not allowed). The popup is mutually exclusive with an existing token.
- **Acceptance:** With a token already rendered, keying `/` in the textarea does not flip `slashOpen` to `true`; no second `SlashCommandPopup` portal mounts.

### 7. History re-edit reconstructs the token

- **Current:** History messages are stored as plain text in `messages.content`. Resend or paste into input shows plain text.
- **Target:** When a message that starts with `/cmd-name ` is loaded into the input via the resend / re-edit path, the leading `/cmd-name` is re-parsed and re-rendered as a token; trailing args remain as plain text. Non-slash inputs pass through unchanged.
- **Acceptance:** A helper `parseInputToTokens(text: string): { token?: { name: string; source: CommandSource }; text: string }` is called on the resend path; if the leading `/cmd-name` matches a known command in the registry, the token is rendered; otherwise the input stays plain text.

### 8. Token rendering applies to both welcome and composer textareas

- **Current:** Two `<textarea>` elements — welcome (`ChatArea.tsx:841`) and composer (`:1127`) — each with its own SlashCommandPopup.
- **Target:** Token rendering works on both textareas with the same visual contract and edit semantics. The token component is shared (or two instances behave identically).
- **Acceptance:** Welcome screen token rendering test passes (Tab on popup → token appears at welcome input); composer token rendering test passes (Tab on popup → token appears at composer input). Visual snapshot diffs are within tolerance.

## Boundaries

**In scope:**

- Renderer-only changes to `ChatArea.tsx` and a new `SlashToken.tsx` component (or equivalent).
- A new `getSlashTokenIcon(source)` helper and a `formatTokenLabel(name)` helper.
- Atomic Backspace behavior at the start-of-input position.
- Click-in-token no-op (selection / caret handling).
- Resend / re-edit path token reconstruction (renderer-only; no DB schema change).
- Both welcome and composer textareas.
- Visual token styling: rounded pill, source-specific icon, Title-Cased name, trailing cursor.

**Out of scope:**

- **LLM payload format change** — preserves v1.1 D-18 design; dispatcher `resolve()` / `dispatch()` and `sendMessage()` payload are unchanged. The `inputVal` literal `/cmd-name args` is the contract.
- **Database schema change** — `messages.content` remains plain text; no new column for `commandRef` metadata.
- **Light theme palette for tokens** — deferred (C-01: 亮色主题 7 色彩色 deferred to v1.2+). 8.1 ships dark theme only.
- **Per-source token background color** — the reference image shows pink for one source; 8.1 uses a neutral token background (e.g., `bg-[var(--color-bg-active)]`) consistent with existing UI surfaces. Source differentiation is via the icon + name, not the background.
- **Mid-input slash rendering** — `请运行 /gsd-fast` mid-text is still plain text (preserves v1.1 D-12 "command at start of message").
- **Multi-token per input** — explicitly excluded (Round 4 Q2: 仅一个 token).
- **Editable token body** — clicking inside the token to edit its name is excluded (Round 4 Q1: 点击中间无效).
- **Clipboard paste token reconstruction** — only the in-app resend / re-edit path reconstructs tokens (Requirement 7). Arbitrary paste with `/cmd-name` is treated as plain text to avoid regex ambiguity in arbitrary clipboard contents.
- **CHANGELOG / 亮色主题 / SLASH-15 (SQLite sessions)** — v1.2+ backlog items, not 8.1.
- **Token animation / transition** — visual polish beyond the static pill; deferred.

## Constraints

- **IME composition is preserved** — the existing 3-ref IME guard (`isComposingRef` + `justFinishedComposingRef` + 200ms timer in ChatArea.tsx:150–152, 648–680) must not regress. Token rendering must not break IME candidate insertion or `compositionend` flows.
- **Shift+Enter newline is preserved** — `e.key === 'Enter' && !e.shiftKey` check in `handleKeyDown` (ChatArea.tsx:733) must continue to work; token rendering does not affect this path.
- **dispatcher contract is preserved** — `inputVal` literal `/cmd-name args` is the contract surface for `dispatcher.resolve()` and downstream `sendMessage()`. No structural change to the input value.
- **Tailwind v4 static class scan** — any new utility classes must be statically present in source (no dynamic concatenation), per Phase 8 D-01..D-04 invariant.
- **Both textareas share the token component** — DRY: a single `SlashToken` component is used by both welcome and composer; no copy-paste.
- **No new dependencies** — phase uses only existing deps (React, lucide-react, Radix Popover, cmdk). No new packages.
- **No DB / IPC / preload changes** — renderer-only.

- [ ] Selecting a slash command via Tab/Enter on the composer renders a token in place of plain text

- [ ] The token label is Title-Cased: `gsd-fast` → `Gsd-Fast`, `goal` → `Goal`
- [ ] After selection, `inputVal` literal state still equals `/cmd-name ` (verified via state query)

- [ ] Resending a history message with leading `/cmd-name` re-renders the token at the start
