## Decisions Made

- **Parser slice length is conditional** (3-way): if `text === '/name'` → slice `len('/name')`; elif `text === '/name '` → slice `len('/name ')`; else (has args) → slice `len('/name')` to keep the leading space. This was a critical correctness fix found during test-driven development — the plan's prose said `text.slice(('/' + c.name).length)` but the acceptance criteria required `/goal ` (with trailing space, no args) to return `text: ''`, which a single-condition slice would have returned as `' '`. The conditional logic handles all three cases correctly.
- **No new dependencies.** All work uses lucide-react (already installed) and the existing SlashCommand + CommandSource types from src/shared/types.ts.
- **iconMap is NOT a default export.** Named export `getSlashTokenIcon` only — matches the dispatcher.ts pattern (named exports for the helper functions).
- **iconMap verified end-to-end, not via unit test.** Per D-08, the 4-test-file budget is exhausted by formatTokenLabel.test.ts + parseInputToTokens.test.ts + Plan 02's SlashToken.test.tsx + Plan 02's SlashToken 5-category visual cases. iconMap's 7-source → 5-icon collapse is exercised through the Plan 02 <SlashToken> live component test, not an isolated unit test.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Fixed parseInputToTokens to consume trailing space when no args are present**
- **Found during:** Task 2 (parseInputToTokens RED phase — `formatTokenLabel` TDD revealed the case-2 test failure)
- **Issue:** The plan's implementation note (`text.slice(('/' + c.name).length)`) would have produced `text: ' '` for input `/goal ` (single trailing space), but acceptance criterion #2 explicitly required `text: ''` for that input. The 3-way matcher in `dispatcher.resolve()` distinguishes `/goal` (no space) from `/goal ` (space) from `/goal args`, so a single slice formula cannot satisfy all three cases.
- **Fix:** Replaced the single slice with a 3-way conditional: `text === '/name' || text === '/name '` → slice to end of text (returns `''`); else → slice `len('/name')` (preserves leading space for the args case). Documented in the implementation with a comment table mapping each matcher condition to its slice length.
- **Files modified:** `src/renderer/src/lib/commands/parseInputToTokens.ts`
- **Verification:** All 11 parseInputToTokens test cases pass; all 18 dispatcher tests pass; all 3 test files (38 tests) pass together.
- **Committed in:** `0b7d1d6` (Task 2 commit)
