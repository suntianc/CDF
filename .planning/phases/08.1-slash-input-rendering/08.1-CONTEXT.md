**Gathered:** 2026-06-05
**Status:** Ready for planning

## Phase Boundary

Renderer-side inline visual rendering of slash command tokens in the Master Agent composer & welcome textareas. When the user selects a slash command via the popup, the input visually displays a styled token (icon + Title-Cased name) in place of the leading `/cmd-name`, but the underlying `inputVal` remains the literal string so the dispatcher contract is preserved. Atomic Backspace + click-no-op on token + history re-parse utility.

Out of scope: dispatcher / IPC / DB / LLM payload changes, light theme, multi-token, editable token body, arbitrary clipboard paste reconstruction.

## Implementation Decisions

### Rendering approach

- **D-01:** **Overlay rendering** — keep `<textarea>` as the input source (IME, selection, copy-paste behavior preserved). Render an absolutely-positioned `<div>` overlay above the textarea, sized to match, that displays the input value with leading tokens rendered as styled pills and the rest as plain text. Textarea text is `color: transparent` so the overlay shows through.
  - **Why:** Two textareas (welcome + composer) already have mature IME composition protection (3 refs + 200ms timer in ChatArea.tsx:150-152, 648-680). contentEditable would force re-testing IME / selection / paste / focus rings. Overlay reuses the existing `<textarea>` and adds a sibling render layer.
  - **Files:** `SlashToken.tsx` (new), `ChatArea.tsx` (overlay markup + sync logic)
  - **Sync:** overlay renders `inputVal` with leading `/cmd-name ` replaced by `<SlashToken ... />` span; trailing text after the token is plain.

### Source icon mapping (5 categories, not 7)

- **D-02:** Collapse 7 `CommandSource` types into 5 visual categories. The 7-source discriminator remains in `SlashCommand.source` (for the popup badge color), but at the input token layer, only 5 icons are used:
  - `system` → `Sparkles` (lucide-react)
  - `skills` (collapses `skill:project` + `skill:global`) → `GraduationCap`
  - `mcp` → `Wrench`
  - `workflow` → `Play`
  - `command` (collapses `cmd:project` + `cmd:system`) → `Terminal`
- **Why:** User explicitly chose 5-category mapping. Two pairs (skills / command) have global vs. project scope differentiation in the popup badge text but not enough visual distinction to warrant separate icons in the input token.
- **File:** `src/renderer/src/lib/commands/iconMap.ts` (new). Helper: `getSlashTokenIcon(source: CommandSource): LucideIcon` returns the icon for any of the 7 sources, with the 2 pair-collapses applied.
- **Tailwind v4 static scan:** all 5 icon names are statically referenced (Phase 8 D-01..D-04 invariant).

### Resend / re-edit path scope (clarification)

- **D-03:** **Parser utility only** — `parseInputToTokens(text: string, registry: SlashCommand[]): { token?: { name: string; source: CommandSource }; text: string }` is exported from a new `src/renderer/src/lib/commands/parseInputToTokens.ts`. No "resend" button or MessageItem edit affordance is added in 8.1.
- **Why:** Codebase has no existing resend / re-edit path (`grep` confirmed: no `resend`, `重发`, `edit message` references in renderer). Adding a new resend UI is its own capability (separate phase). 8.1 ships the parser + wires it into the overlay re-render path so the utility is ready for future resend features.
- **What this means for the overlay:** When the overlay mounts OR `inputVal` is set externally (e.g., via paste handler or future resend), the overlay re-runs `parseInputToTokens()` to determine if the leading text matches a known command and re-renders the token.

### Atomic Backspace + click-no-op wiring

- **D-04:** **Event interception** in `ChatArea.tsx`:
  - **Backspace atomicity:** `handleKeyDown` (existing) is extended. When the caret sits at `position len('/cmd-name ')` (i.e., immediately after the trailing space of a rendered token), pressing `Backspace` triggers `e.preventDefault()` + `setInputVal('')` (deletes the whole `/cmd-name ` prefix in one event).
  - **Click no-op:** Each `<SlashToken>` span has `onMouseDown={(e) => e.preventDefault()}` to prevent the textarea from losing focus or moving the caret inside the token. This keeps the token as an atomic visual unit.
- **Why:** CSS-only approach (`user-select: none` + `pointer-events: none`) would block caret interaction entirely — including legitimate focus restoration. Event interception gives surgical control and integrates with the existing `handleKeyDown` and IME guards.
- **Coexists with:** IME composition guard (Backspace during IME composition is handled by `isComposingKeyEvent` before this check); Shift+Enter newline path (unaffected).

### Title-Case name formatting

- **D-05:** `formatTokenLabel(name: string): string` — first letter of every hyphen-separated segment is uppercased; hyphens preserved. Examples: `gsd-fast` → `Gsd-Fast`; `goal` → `Goal`; `code-review` → `Code-Review`; `run-skill-generator` → `Run-Skill-Generator`.
- **Why:** Locked in SPEC R3. Implementation: split on `-`, map first-letter-uppercase, rejoin. Unit test: `formatTokenLabel('gsd-fast') === 'Gsd-Fast'`, `formatTokenLabel('goal') === 'Goal'`.
- **File:** `src/renderer/src/lib/commands/formatTokenLabel.ts` (new).

### Component structure (Claude's discretion)

- **D-06 (Claude's discretion):** New file `src/renderer/src/components/SlashCommand/SlashToken.tsx` exporting a single `<SlashToken>` React component. Props: `name: string`, `source: CommandSource`, `data-testid?: string`. Renders a `<span>` with `data-slash-token` attribute, inline icon (size 12px), Title-Cased name, and Tailwind classes for the pill (rounded-full background, padding, gap). Both welcome and composer overlays consume this component.
- **D-07 (Claude's discretion):** Overlay markup in `ChatArea.tsx` wraps each `<textarea>` in a relative-positioned container; a sibling `<div>` with the same width/height sits on top, rendered conditionally only when `slashToken` is non-null. Z-index: 1 (token pill) vs 0 (textarea), so textarea retains focus.

### Testing strategy (Claude's discretion)

- **D-08 (Claude's discretion):** Three new test files:
  1. `src/renderer/src/components/SlashCommand/SlashToken.test.tsx` — visual snapshot of `<SlashToken name="gsd-fast" source="skills" />`, plus 4 source-iteration tests (one per category other than system).
  2. `src/renderer/src/lib/commands/parseInputToTokens.test.ts` — 6 unit cases: leading `/goal fix login`, leading `/unknown abc` (no match), non-slash text, empty string, multiple slashes (only first parses), trailing space edge case.
  3. `src/renderer/src/lib/commands/formatTokenLabel.test.ts` — 4 unit cases (single word, hyphenated, multi-hyphen, edge cases).
- **Regression test:** `dispatcher.test.ts` (existing) — assert `resolve('/goal fix login', ...)` still returns the same `SystemSilent` plan; ensures the literal-string contract is preserved.
