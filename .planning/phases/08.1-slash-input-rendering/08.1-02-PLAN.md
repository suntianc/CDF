    - `data-slash-token=""` — semantic marker for the overlay in Plan 03 (used to find the rendered token for caret-position calculations if needed in the future; harmless now).
    - `contentEditable={false}` — defensive: even if the parent accidentally renders inside a contentEditable parent, the token refuses to be edited.
    - `onMouseDown={(e) => e.preventDefault()}` — D-04 click-no-op. Prevents the textarea from losing focus or moving the caret inside the token.
    - `className={cn(
        'inline-flex items-center gap-1 align-middle',
        'rounded-full px-1.5 py-0.5',
        'bg-[var(--color-bg-active)] border border-[var(--color-border)]',
        'text-[11px] font-medium text-[var(--color-text-primary)]',
        'select-none whitespace-nowrap',
        'mx-0.5',
        'pointer-events-auto'
      )}` — neutral surface (SPEC out-of-scope: no per-source background). All Tailwind utility classes are HARDCODED (Phase 8 D-01..D-04 invariant: no dynamic concatenation). The pill is `inline-flex` so it sits inside the overlay's text flow next to the trailing plain text. `pointer-events-auto` is REQUIRED (overrides the overlay parent's `pointer-events-none` set in Plan 03) so the `onMouseDown` handler below can fire and call `preventDefault` to block caret entry. Without `pointer-events-auto`, the click would pass through to the textarea behind and the caret would land inside the token — which is exactly the SPEC R5 violation this plan prevents.

    Inside the span:

    - The icon: `<Icon className="w-3 h-3 text-[var(--color-text-secondary)]" aria-hidden="true" />`. Size 12px, secondary text color so the icon is visually subordinate to the name (per the reference image's cube + "Gsd Fast" emphasis on the name).
    - The label: `<span>{formatTokenLabel(name)}</span>` — the Title-Cased display.

    Do NOT add a trailing space or `&nbsp;` inside the span — the chat text node after the token (rendered in Plan 03) is responsible for the gap.

    Do NOT add a focus ring, hover state, or animation — the SPEC Out-of-Scope list explicitly excludes "Token animation / transition" and the neutral surface is intentional. A simple static pill is the target.

    No state, no `useEffect`, no `useRef`. This is a pure presentational component.
