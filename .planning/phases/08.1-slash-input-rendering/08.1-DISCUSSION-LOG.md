## Atomic Backspace + click-no-op wiring

| Option | Description | Selected |
|--------|-------------|----------|
| 事件拦截（onKeyDown + onMouseDown preventDefault） | onKeyDown 检测 caret 位置在 token 尾后时拦截 Backspace；onMouseDown 在 token 上 preventDefault 阻止 caret 进入。逻辑在 ChatArea.tsx 集中管理 | ✓ |
| 纯 CSS 机制（user-select + pointer-events） | Token 容器加 `user-select: none` + `pointer-events: none`；caret 只看到 textarea 本身。Backspace 仍用 onKeyDown 拦截一行（但没 caret 检测，纯状态推断） | |
| 封装为 useSlashToken() hook（隔离） | 全部抽到 useSlashToken() hook 中（state: 是否有 token、args；actions: handleBackspace, handleMouseDown）。ChatArea 只调 hook 接口 | |

**User's choice:** 事件拦截（onKeyDown + onMouseDown preventDefault）
**Notes:** Event interception gives surgical control and integrates with the existing `handleKeyDown` (ChatArea.tsx:721) and IME guards. Backspace atomicity is detected via caret position: when caret sits at `len('/cmd-name ')` after a rendered token, a single Backspace clears the entire prefix. Click no-op is `onMouseDown={(e) => e.preventDefault()}` on the SlashToken span.
