# Phase 08.2 Plan 01: Foundation — body load + frontmatter + IPC + argSubstitution + dispatcher body integration

**One-liner:** Lazy .md body load via `commands:readBody` IPC with path-traversal guard, real YAML frontmatter parsing (yaml@2.9.0), `$ARGUMENTS`/`$0`/`$N`/`$name` substitution, and dispatcher PluginRewrite path that sends the substituted body as the user message (command name does not appear).

## Task 3: commands:readBody IPC + argSubstitution + dispatcher PluginRewrite integration

**IPC handler (src/main/ipc-handlers.ts):**
- `commands:readBody(bodyPath: string)` → `{ body: string; mtimeMs: number }`
- ASVS V5.1.3 path-traversal guard: allowlist = `~/.cdf/commands/` + every registered project's `.cdf/commands/` (queried from `db.projects`); `path.resolve(bodyPath)` must start with one of the allowlist prefixes
- Defensive: typeof + length 0..1024 guard; `console.warn` on path-traversal attempt; never throws; returns `{ body: '', mtimeMs: 0 }` on any failure

**Preload bridge (src/preload/index.ts):**
- `commands.readBody(bodyPath)` added to the existing `commands` namespace

**chokidar unlink tests (Issue 7 probe disposition):**
- 2 new unlink test cases (system + project scope) verifying onChange + commands:changed IPC push on unlink
- No `removeByPath` function added (registry is stateless; the existing `commands:changed` IPC push is the recovery mechanism)

**argSubstitution utility (src/renderer/src/lib/commands/argSubstitution.ts — NEW):**
- Pure string transform: `String.prototype.replaceAll` + `RegExp.replace` only
- 3 steps: $ARGUMENTS verbatim → $0..$N positional → $name via `arguments` declaration
- Out-of-range $N keeps literal; undeclared $name keeps literal
- No eval, no Function, no child_process (V10 verified: `grep -cE "\beval\b|new Function\(|child_process" = 0` in the file)
- 8 co-located test cases: $ARGUMENTS / $0 / multi $N / $N out-of-range / $name declared / $name undeclared / CJK args / no placeholders

**Dispatcher PluginRewrite integration (src/renderer/src/lib/commands/dispatcher.ts):**
- If `plan.command.bodyPath` is set:
  1. `await window.electronAPI.commands.readBody(bodyPath)` → body
  2. If body is non-empty: `substituteArgs(body, { args: plan.args, arguments: plan.command.frontmatter?.arguments })`
  3. `await sendMessage(projectId, substituted, hasOverrides ? overrides : undefined)` — body replaces user message (D-03: command name does not appear)
- Fall-through: when no bodyPath or empty body, existing prompt-rewrite path is used
- D-09: `frontmatter.allowedTools` → `overrides.allowedTools` when non-empty array

**Tests:**
- `dispatcher.test.ts`: 21 cases (18 original + 3 new for bodyPath/no-bodyPath/allowedTools)
- `argSubstitution.test.ts`: 8 cases (all green)
