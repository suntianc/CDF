# Phase 08.2 — UI Design Contract

> Visual and interaction contract for Phase 08.2 (slash command execution principle optimization). Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Scope:** Three distinct UI surfaces per F-01 (不统一为一种呈现):
> 1. `/goal` 系统气泡 (持续状态机) — C-1d
> 2. `/context` 弹窗 modal (token breakdown) — C-2b
> 3. `/plan` 弹窗 modal (Codex 弹窗 + modify loop) — C-3a
> 4. `/context` 常驻按钮 (dual entry) — C-2d

## State Machines

### /goal bubble state machine (C-1)

```
[absent] ──(user /goal X)──> [working]
[working] ──(judge: yes)──> [done]
[working] ──(judge: no + reason)──> [working]  (counter increments, reason sent to agent as user msg)
[working] ──(judge parse error)──> [failed]
[working] ──(turn count = 20)──> [paused]
[done / failed / paused] ──(user /goal clear)──> [absent]
```

| State | Visual | Color | Icon |
|-------|--------|-------|------|
| working | pulsing border (1s ease-in-out) | `accent` (#7c3aed) | `◎` (Unicode U+25CE) |
| done | static, `success` background tint | `success` (#22c55e) | `✅` |
| failed | static, `danger` background tint | `danger` (#ef4444) | `⚠️` |
| paused | static, `warning` background tint | `warning` (#f59e0b) | `⏸` |

### /plan modify loop state (C-3)

```
[closed] ──(user /plan X)──> [generating]
[generating] ──(first chunk arrives)──> [reviewing]
[reviewing] ──(user clicks 修改 + submits)──> [modifying]  (n += 1)
[modifying] ──(new plan arrives)──> [reviewing]  (n += 1)
[reviewing / modifying] ──(user clicks 立即执行)──> [executing]  (sends user msg, closes)
[executing] ──(modal close)──> [closed]
[any state] ──(n = 20)──> [cap-reached]  (only 立即执行 button enabled)
```

## Notes for Planner / Executor

1. **No shadcn CLI initialization.** This project uses manual Radix wrapper pattern (per Phase 6 D-19). The `shadcn_initialized: false` frontmatter is correct and permanent.

2. **3 surfaces, not 1.** F-01 explicitly rejected unifying the 3 system command UIs. Each surface has:
   - Distinct accent color (per F-01)
   - Distinct component (Bubble vs Modal vs Modal)
