## Decisions Made

- **C3-05 default:** PlanMode.popupOpen defaults to TRUE in resolve(). Callers that need the legacy toast path must explicitly pass `popupOpen: false`. This makes the new popup the default UX (Codex-style) without breaking the existing dispatcher tests in the P1 plan.
- **P7 cap location:** 20-turn cap enforced in store's `startModify` action (not in component). This guarantees the cap is honored even if PlanPopup is bypassed (e.g., direct programmatic invocation). Status='cap-reached' is a visible UI state.
- **P11 mitigation:** store tracks `currentRequestId` and falls back to `useSessionStore.getState().streamingMessageId` if null. `cancelCurrent()` and `close()` both call stopChat to prevent orphan LLM runs.
- **D-09 UI scope:** user-invocable filter is applied in `displayCommands` useMemo (renderer-side, before normalization/filter pipeline). System-hardcoded commands have no frontmatter, so they default to invocable — preserves Phase 7 `/goal` / `/context` / `/plan` behavior.
- **Issue 4 fix:** runtime.ts is NOT in files_modified for this plan. The planOnly tool gate is already enforced in Phase 7 (per `runtime.ts:490-496, 610`); this plan only writes a regression test, not runtime logic.
- **Issue 3 fix:** pre-execution probe (grep runtime.ts) determines gate status BEFORE writing the test. Probe result = ENFORCED, so the test file writes real `it` cases (3 cases, all pass). If a future refactor weakens the gate, this test fails immediately as a clear signal.
