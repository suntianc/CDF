types：
- JudgeStatus = 'idle' | 'judging' | 'satisfied' | 'unsatisfied' | 'failed' | 'paused'
- interface JudgeDecision { satisfied: boolean; reason: string }
- interface JudgeOptions { maxTurns?: number; model?: string }  // maxTurns default 20
- interface GoalJudgeStatusEntry { status: JudgeStatus; iteration: number; startedAt: number; reason?: string }

主要导出（factory 模式 — Issue 8 修复，测试可独立 reset）：
- export function createGoalJudge(): GoalJudgeInstance
  - 返回 { startGoalJudgeLoop, stopGoalJudgeLoop, useGoalJudgeStatus, _internal }
  - 内部 `const subs = new Map<string, { unsubscribe: () => void; controller?: AbortController }>()` 闭包持有（非模块级 singleton）
  - 便于 RTL 测试独立创建实例，避免模块级状态泄漏
- export const goalJudge: GoalJudgeInstance = createGoalJudge()  // 默认 singleton
- 别名导出：`export const startGoalJudgeLoop = goalJudge.startGoalJudgeLoop` 等
- async function startGoalJudgeLoop(sessionId: string, goal: string, options: JudgeOptions = {}): Promise<void>
  - 若 subs 已有 sessionId entry：先 stopGoalJudgeLoop（防重入）
  - 写入 setGoalJudgeStatus(sessionId, { status: 'idle', iteration: 0, startedAt: Date.now(), reason: undefined })
  - 取 projectId from useProjectStore；无 → console.warn + return
  - 首轮：`useSessionStore.getState().sendMessage(projectId, goal)`（不带 overrides，平等 user message）
  - 调内部 `subscribeRunCompleted(sessionId, onRunCompleted)` 注册回调；保存 unsubscribe 到 subs
  - onRunCompleted 内部：
    - setGoalJudgeStatus(sessionId, { status: 'judging' })
    - try { const decision = await judgeOnce(sessionId, goal); ... } catch (e) { setGoalJudgeStatus(sessionId, { status: 'failed', reason: 'judge 失败：' + (e.message || 'unknown') }); unsubscribe; return }
    - if decision.satisfied: setGoalJudgeStatus(sessionId, { status: 'satisfied', reason: decision.reason }); unsubscribe; return
    - else:
      - const next = getGoalJudgeStatus(sessionId).iteration + 1
      - if next >= (options.maxTurns ?? 20): setGoalJudgeStatus(sessionId, { status: 'paused', reason: '已达 ' + maxTurns + ' 轮上限' }); sonner.warning('/goal 暂停', { description: '已达 ' + maxTurns + ' 轮上限。输入 /goal 清空。' }); unsubscribe; return
      - setGoalJudgeStatus(sessionId, { status: 'unsatisfied', iteration: next, reason: decision.reason })
      - await useSessionStore.getState().sendMessage(projectId, '继续：' + decision.reason)
- async function stopGoalJudgeLoop(sessionId: string): Promise<void>
  - sub = subs.get(sessionId); if sub: sub.unsubscribe(); sub.controller?.abort(); subs.delete(sessionId)
  - clearGoalJudgeStatus(sessionId)
- function useGoalJudgeStatus(sessionId: string | null): { status; iteration; startedAt; reason?, goal }
  - React hook，subscribe useSessionStore.goalJudgeStatus 用 selector + sessionGoals；return { status, iteration, startedAt, reason?, goal }

judge call 实现细节：
- 内部辅助 async function judgeOnce(sessionId, goal): Promise<JudgeDecision>
- 取最近 N=8 轮 message 文本（不含 tool / system / goal-system bubble）从 sessionStore.messages
- 构造 judge prompt：
  ```
  You are a strict judge. Evaluate whether the following completion condition is met by the AI agent's recent work.

  Condition: ${goal}

  Recent conversation (last 8 turns):
  ${recentTurns}

  Respond with JSON ONLY, no preamble, no markdown fence. Schema:
  { "satisfied": boolean, "reason": "1-2 sentence rationale in 简体中文" }
  ```
- 调用 `await window.electronAPI.llm.chat({ ... 复用 main agent 同 provider/model, history: [judgePrompt as user], system: '' })`（具体 payload 形状由 chat IPC 现状确定）
- 收到完整响应后：先剥离 `<think>...</think>` 块（regex `/<think>[\s\S]*?<\/think>/g` 替换为 ''）；然后 trim；然后 JSON.parse；失败抛 Error
- judge 自身的 chat 调用必须**不**经过 sessionStore.sendMessage（避免污染对话流 + 触发递归 judge）；直接调 electronAPI.llm.chat 或新增一个不入 sessionStore 的 thin wrapper
