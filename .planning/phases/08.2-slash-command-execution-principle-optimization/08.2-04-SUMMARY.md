## 11-Category Breakdown — Real vs Placeholder (Issue 1 强制要求)

| # | Category | v1.1 Status | Source / Computation |
|---|----------|-------------|----------------------|
| 1 | System prompt | **placeholder (0)** | No source exposed; v1.2 推 |
| 2 | System tools | **placeholder (0)** | Built-in tool schema not enumerated; v1.2 推 |
| 3 | MCP tools | **real** | `loadMcpTools` → sum JSON.stringify(tool.schema).length × 0.25 |
| 4 | Workflows | **real** | DB `workflows` table → SUM(LENGTH(graph_data)) |
| 5 | Custom agents | **placeholder (0)** | Subagent definitions not exposed; v1.2 推 |
| 6 | Memory files | **placeholder (0)** | MEMORY.md / CLAUDE.md not wired; v1.2 推 |
| 7 | Skills | **real** | `listPhysicalSkills` → sum SKILL.md file sizes |
| 8 | Messages | **real (alias of conversation)** | DB `messages` table → SUM(LENGTH(content)) |
| 9 | Project command bodies | **real** | Sum `<projectPath>/.cdf/commands/*.md` byte sizes × 0.25 |
| 10 | Free space | **computed** | max(0, contextLimit - total - autocompactBuffer) |
| 11 | Autocompact buffer | **computed** | Math.ceil(contextLimit × 0.15) |

**Issue 1 acknowledgment (CONTEXT.md §C2-01):** The 4 placeholder categories (`systemPrompt` / `systemTools` / `customAgents` / `memoryFiles`) default to 0 and emit a `console.warn` marker at compute time. The user sees 7 real numbers + 4 zeros, with the modal's data-testid rows rendering all 11. This is **not** a C2-01 "完整版 breakdown" promise — it's a v1.1 MVP that hands the full UI surface to the user; v1.2 closes the estimation gap. Acceptance criteria for "11 categories present" are satisfied; the 4 placeholders are documented as such in code comments.

| Field | Where enforced | Mechanism | Test |
