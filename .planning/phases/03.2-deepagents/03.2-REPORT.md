### 模块 5：持续优化

**复核文件:** `src/main/deepagent/mcp-connector.ts`, `src/shared/types.ts`, `src/renderer/src/components/PluginsPanel/PluginsPanel.tsx`
**检查项:** 12 项 | **通过:** 10 项 | **待评估:** 2 项 | **修复:** 1 项

**核心发现:**
- **已修复 (W-09):** MCP 传输类型从 `sse` 更新为 `http`，利用 @langchain/mcp-adapters 的自动回退机制（先尝试 Streamable HTTP，后回退 SSE）
- **无需迁移 (W-10):** `streamEvents` v3 是 deepagents 的高级投影式流式接口，已内置子代理事件隔离（`run.subagents`）、reasoning token 支持（`msg.reasoning`）、工具调用事件（`run.toolCalls`）。LangGraph 的 `stream()` 是底层 API，迁移意味着从高级接口降级到低级接口（Spike 001 验证）
- **不迁移 (W-11):** `initChatModel` 统一模型创建与项目自定义 patch 逻辑（patchOpenAIReasoning、patchMiniMaxAssistantRole）不兼容，直接实例化 + 手动 patch 是更可控方案
- **可优化 (W-12):** todo 轮询机制（500ms 间隔）可用 `run.values`（AsyncIterable）替代。`run.values` 在每个步骤后产出状态快照，通过检测 `todos` 字段变化实现事件驱动推送，消除轮询延迟。需验证 `run.values` 产出的 state 中是否包含 `todos`
- Skills permissions deny 规则作为纵深防御保留，与 CompositeBackend 互补

**代码变更:**
- `mcp-connector.ts`: 传输类型 `'sse'` -> `'http'`
- `shared/types.ts`: MCPServer.server_type 添加 `'http'` 选项
- `PluginsPanel.tsx`: 表单类型迁移为 `'http'`，向后兼容 `'sse'`

**Commit:** `15a3d2c`
