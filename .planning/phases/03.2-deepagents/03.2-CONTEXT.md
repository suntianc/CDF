# Phase 03.2: deepagents - Context

**Gathered:** 2026-05-27
**Status:** Ready for planning

<domain>
## Phase Boundary

基于 deepagents.js 官方文档（JS 版本），对当前产品进行系统性复核。按模块逐步筛查，检查 API 兼容性 + 最佳实践，发现问题直接在本阶段修复。每个模块筛查完毕后提供报告供用户分析确认，确认后再处理下一个模块。

复核范围：Runtime 创建与配置、子Agent 委托机制、MCP 集成、Skills 系统。

</domain>

<decisions>
## Implementation Decisions

### 1. 复核范围（全部模块）
- **D-01:** 复核覆盖所有 deepagents 集成模块：Runtime 创建与配置、子Agent 委托机制、MCP 集成、Skills 系统
- **D-02:** 不遗漏任何模块，确保全面覆盖

### 2. 检查标准
- **D-03:** 检查深度为「API 兼容性 + 最佳实践」
  - API 签名、参数、返回值与官方文档一致性
  - 是否使用了推荐模式、是否遗漏了最佳实践
  - 不做性能调优或错误处理模式的全面审查

### 3. 问题处理策略
- **D-04:** 发现的问题在本阶段直接修复，不记录到 backlog
- **D-05:** 按模块一步一步筛查、修复
- **D-06:** 每个模块筛查完毕后提供一份报告给用户分析
- **D-07:** 用户确认无问题后，再进行下一个模块的处理

### 4. 参考文档
- **D-08:** 使用 deepagents.js 官方文档（JavaScript/TypeScript 版本）
- **D-09:** 文档基础 URL: `https://docs.langchain.com/oss/javascript/deepagents/`
- **D-10:** 不参考 Python 版本文档（概念可能有差异）

### 5. 筛查顺序
- **D-11:** 按以下顺序逐模块筛查：
  1. **任务规划与协调** — Runtime 创建、Agent 配置、createDeepAgent 参数
  2. **环境操作与上下文管理** — 上下文窗口管理、总结策略、会话持久化
  3. **子代理委派** — task(name, task)、subagentIds、responseFormat、异常包装
  4. **人机协同** — approval_required 机制、用户确认流程
  5. **持续优化** — 整体架构一致性、代码质量、模式统一性

### 6. 报告格式
- **D-12:** 每个模块的报告包含：
  - 检查项清单（与官方文档对比）
  - 发现的问题（严重程度：critical / warning / info）
  - 建议的修复方案
  - 修复状态（待确认 / 已修复 / 不需要修复）

### Claude's Discretion
- 具体的 deepagents.js 文档页面 URL 由 researcher agent 在研究阶段确定
- 每个模块的检查粒度由 planner 根据模块复杂度决定
- 修复方案的技术细节由 executor 在实现阶段决定

</decisions>
