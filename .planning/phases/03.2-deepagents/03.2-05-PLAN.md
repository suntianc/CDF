**检查项 1: MCP 传输类型 SSE → http**（per 研究问题 5.1）
当前 mcp-connector.ts 第 49 行使用 `transport: 'sse'`。MCP 规范 2025-03-26 已将 SSE 替换为 Streamable HTTP。

修复方案：
1. 检查 `@langchain/mcp-adapters` 是否支持 `transport: 'http'` 类型
2. 如果支持，将 mcp-connector.ts 第 49 行的 `transport: 'sse'` 改为 `transport: 'http'`
3. 同时更新 shared/types.ts 中 MCPServer 的 `server_type` 类型定义，添加 `'http'` 选项
4. 检查前端 UI 中 MCP 服务器配置是否需要更新传输类型选项

**检查项 2: streamEvents v3 vs stream() API 评估**（per 研究问题 5.7）
当前 llm.ts 第 349 行使用 `agent.streamEvents(nextInput, { version: 'v3' })`。官方文档推荐使用 `agent.stream()` 配合 `streamMode` 和 `subgraphs: true`。

评估内容：
1. 对比两种 API 在子代理事件获取上的行为差异
2. 检查当前代码中 `run.messages` 和 `run.toolCalls` 的使用是否在 `stream()` API 中有对应
3. 评估迁移工作量和风险
4. 如果迁移风险较高，在报告中标注为「待评估」而非直接修改

**检查项 3: todo 轮询机制**（per 研究问题 5.9）
llm.ts 第 293-312 行使用 500ms 间隔轮询 todo 状态。检查：
1. deepagents 内置的 TodoListMiddleware 是否提供事件流推送 todo 更新
2. 如果提供，自定义轮询机制可以移除或降级为备用方案
3. 如果不提供，保留轮询机制

**检查项 4: MCPServer 类型定义更新**
如果 MCP 传输类型更新为 http，需要同步更新 shared/types.ts 中的 MCPServer 接口：
- `server_type: 'stdio' | 'sse'` → `server_type: 'stdio' | 'sse' | 'http'`

**检查项 6: Skills 系统 permissions 配置**（per 研究问题 5.6）
