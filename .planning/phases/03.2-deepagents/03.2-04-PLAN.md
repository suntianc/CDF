按以下步骤逐一检查并修复（per D-03, D-04, D-05）：

**检查项 1: respond 决策类型缺失**（per 研究问题 4.4 — 主要问题）
当前 shared/types.ts 第 160 行 `AgentApprovalDecisionType = 'approve' | 'reject' | 'edit'` 缺少第 4 种决策类型 `respond`。官方文档定义了 4 种决策类型：
- `approve` — 使用原始参数执行
- `edit` — 修改参数后执行
- `reject` — 跳过执行
- `respond` — 返回用户消息作为工具结果

修复方案：
1. 在 shared/types.ts 中将 `AgentApprovalDecisionType` 更新为 `'approve' | 'reject' | 'edit' | 'respond'`
2. 检查 llm.ts 中 approval 决策处理逻辑（第 729-735 行），确认是否需要添加 respond 的处理分支
3. 检查 runtime.ts 中 DEFAULT_INTERRUPT_ON 配置（第 53-57 行），确认是否需要在某些工具中允许 respond 决策

**检查项 2: interruptOn 配置完整性**（per 研究问题 4.1）
当前 DEFAULT_INTERRUPT_ON 只配置了 write_file、edit_file、delete_file 三个工具。检查：
1. 是否还有其他工具需要 interruptOn 配置（如 bash、fetch）
2. 官方文档中 interruptOn 的默认行为（`true` = 默认中断）是否适用于未列出的工具

**检查项 3: approval 处理流程中的 respond 处理**
llm.ts 第 729-735 行的 approvalStatus 判断逻辑：
```typescript
const approvalStatus = resolution.decisions.some((decision) => decision.type === 'edit')
  ? 'edited'
  : resolution.decisions.every((decision) => decision.type === 'approve')
    ? 'approved'
    : 'rejected';
```
这个逻辑没有处理 respond 类型。需要添加：
- 当所有决策都是 respond 时，approvalStatus 应该是什么
- respond 决策的 message 字段如何传递给 deepagents

**检查项 4: Command resume 格式**
确认 `new Command({ resume: { decisions: resolution.decisions } })` 的 decisions 格式与 deepagents 期望的格式一致。特别是 respond 类型的决策格式。
